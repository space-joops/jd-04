## A. TypeScript 한 장 요약

```ts
// 변수
const x = 3;            // 재할당 X (기본)
let y = 3;              // 재할당 O

// 타입
let s: string;                 let n: number;         let b: boolean;
let arr: number[];             let obj: { a: number }; let maybe: string | null;
type Id = string;              type Fmt = "utc" | "device" | "home";  // 리터럴 유니온

// 없음 다루기
a ?? b        // a가 null/undefined면 b   (파이썬 a if a is not None else b)
a?.b          // a가 없으면 undefined (터지지 않음)
a?.b?.()      // 있으면 호출

// 함수
function f(a: number): number { return a; }
const g = (a: number): number => a;          // 화살표 (파이썬 lambda 강화판)
const h = (a: number) => { return a + 1; };  // 여러 줄

// 삼항
조건 ? 참일때 : 거짓일때        // 파이썬: 참일때 if 조건 else 거짓일때

// 구조분해 (파이썬 언패킹)
const { a, b } = obj;          const [first, ...rest] = arr;

// 전개 (spread)
const merged = { ...base, extra: 1 };    const arr2 = [...arr, 4];

// 템플릿 문자열
`점수 ${n}점`                   // 파이썬 f"점수 {n}점"

// 타입 마법 (이 프로젝트 i18n)
type DictKey = keyof typeof en;         // 객체 en의 모든 키를 타입으로
type Dict = Record<DictKey, string>;    // 모든 키 → string 강제 (빠지면 컴파일 실패)
```

**함정**: `==` 금지 → `===` 만. `null`≠`undefined`. `number`엔 int/float 구분 없음.

## B. React 훅 한 장 요약

```tsx
// 상태 (바꾸면 리렌더)
const [n, setN] = useState(0);
setN(5);                        // 값으로
setN(x => x + 1);               // 이전 값 기반 (안전)

// 부수 효과 (렌더 후 실행 + 정리)
useEffect(() => {
  const id = setInterval(tick, 1000);
  return () => clearInterval(id);   // 정리: 언마운트/재실행 전 (누수 방지 필수!)
}, [dep]);                          // []=마운트1회, [x]=x바뀔때, 생략=매렌더

// 참조 (바꿔도 리렌더 X — 게임 상태·DOM)
const ref = useRef(null);       // ref.current
<canvas ref={ref} />

// 함수 정체성 고정
const fn = useCallback(() => {...}, [deps]);

// 외부 저장소 구독 (localStorage 등)
const v = useSyncExternalStore(subscribe, getSnapshot, getServerSnapshot);
```

**규칙**: 훅은 컴포넌트 **최상단에서만**(조건문·반복문·중첩 함수 안 X). 직접 대입 금지
(`setX` 써라). 컴포넌트는 **대문자**로 시작.

**서버 vs 클라이언트 컴포넌트**:
- 기본 = 서버(빠름, JS 번들 X, 훅·이벤트·브라우저API X).
- `"use client";` = 클라이언트(훅·이벤트·`window`·`localStorage` O).

## C. JSX 빠른 참조

```tsx
<p className="..." style={{ color: "#fff" }}>텍스트 {변수}</p>  // class→className, style은 객체
<img src="..." alt="..." />              // 자기닫기 태그 필수
<>...</>                                 // 프래그먼트 (감쌀 태그 없이 여러 개)
{cond && <X/>}                           // cond가 참이면 렌더
{cond ? <A/> : <B/>}                     // 삼항 렌더
{list.map(x => <li key={x.id}>{x.name}</li>)}   // 반복 (key 필수!)
<button onClick={() => f()}>클릭</button>       // 이벤트 (클라 컴포넌트만)
return null;                             // 아무것도 안 그림
```

## D. CSS / Tailwind 빠른 참조

```
박스 모델(안→밖):  content → padding(안여백) → border(테두리) → margin(바깥여백)

flexbox:
  flex flex-col        display:flex; 세로
  items-center         align-items:center (교차축 가운데)
  justify-between      주축 균등 분배
  gap-4                자식 간격 1rem

크기·색·간격 (숫자 4 = 1rem = 16px):
  p-4 px-6 py-2        padding
  m-2 mt-auto          margin
  text-sm text-4xl     글자 크기
  text-white text-gray-400   글자 색
  bg-white/10          배경 (투명도 10%)
  border-2 border-white/40   테두리

반응형 (모바일 퍼스트):
  text-4xl md:text-6xl       기본 4xl, 768px+ 에서 6xl
상태:
  hover:bg-white/10  focus-visible:scale-105

위치·쌓임:  relative  absolute  fixed  z-10  min-h-dvh
```

순수 CSS가 필요하면 `globals.css`(예: `@keyframes sw-crawl`). 동적 값은 `style={{}}`
인라인(클래스를 이김).

## G. 용어 사전

| 용어 | 뜻 |
|---|---|
| **DOM** | 브라우저가 HTML을 표현하는 객체 트리. React가 대신 조작. |
| **JSX** | HTML처럼 생긴 JS 문법. 빌드 시 함수 호출로 변환. |
| **컴포넌트** | 화면 조각을 반환하는 함수(대문자 시작). |
| **props** | 컴포넌트에 넘기는 읽기 전용 인자. |
| **훅(hook)** | `use`로 시작하는 React 함수(상태·효과·참조). |
| **리렌더** | 상태 변경 시 컴포넌트 함수를 다시 실행. |
| **재조정(reconciliation)** | React가 최소 DOM 변경을 계산하는 과정. |
| **클로저** | 함수가 바깥 변수를 기억하는 것. 게임 상태의 뼈대. |
| **서버 컴포넌트** | 서버에서 실행, HTML만 전송(기본). |
| **클라이언트 컴포넌트** | `"use client"`, 브라우저에서 실행. |
| **하이드레이션** | 서버 HTML에 브라우저 JS가 "생명을 불어넣는" 과정. |
| **rAF** | `requestAnimationFrame`. 화면 갱신 직전 콜백. |
| **dt** | delta time. 프레임 간 시간(초). 이동은 `값*dt`. |
| **DPR** | device pixel ratio. CSS px당 물리 픽셀 수. |
| **PWA** | 설치·오프라인 가능한 웹 앱. |
| **서비스 워커** | 요청을 가로채는 브라우저 프록시(캐시). |
| **SWR** | stale-while-revalidate. 캐시 즉답 + 백그라운드 갱신. |
| **OG (OpenGraph)** | 링크 공유 시 뜨는 미리보기 메타데이터. |
| **metadataBase** | OG 이미지 절대 URL의 기준 도메인. |
| **RLS** | Row Level Security. DB 행 단위 권한. |
| **RPC** | 원격 프로시저 호출. 검증된 쓰기 통로. |
| **PostgREST** | PostgreSQL을 자동 REST API로 노출(Supabase). |
| **Context** | React의 전역 상태 공유 메커니즘. |
| **tree-shaking** | 빌드 시 미사용 코드 제거. |
| **semver** | `^5`(마이너 허용), `~5.1`(패치만) 버전 규칙. |

## H. 이 프로젝트 3대 원칙 (모든 코드의 배경)

1. **초당 60번 변하는 건 캔버스, 가끔 변하는 것만 React.** (성능 아키텍처)
2. **부가 기능이 앱을 죽이면 안 된다.** (try-catch·null·env 가드 — 견고성)
3. **에셋 파일 0개 — 모든 그림·소리는 코드로.** (fillRect·오실레이터·ImageResponse)
