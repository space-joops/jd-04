# 🚀 5일 웹 프론트엔드 마스터 코스 — 파이썬 데이터 엔지니어를 위한 속성 심화

> **누구를 위한 문서인가**
> 파이썬 백엔드·데이터 엔지니어링은 능숙하지만, 프론트엔드는 "낯선 대륙"인 당신.
> 이 코스는 그 배경을 **약점이 아니라 지렛대**로 씁니다. 당신은 이미 타입 시스템,
> 비동기, 데이터 파이프라인, 상태 관리, 배포를 압니다. 우리는 그 지식을 웹으로
> **번역(translate)**할 뿐입니다.
>
> **교재는 이 저장소 자체(SPACE JOOPS)입니다.** 추상적 예제가 아니라, 실제로
> 배포되어 돌아가는 프로덕션 게임의 코드를 뜯어 봅니다. 모든 개념에는 "이 프로젝트
> 어느 파일 몇 줄"이라는 좌표가 붙습니다. 문서를 읽다 파일을 열면, 똑같은 코드가 있습니다.

## 이 코스가 다루는 실제 기술 스택

이 프로젝트에 **실제로** 쓰인 것들입니다. 유행어 나열이 아니라, 프로덕션에서 검증된 조합.

| 레이어 | 기술 | 버전 | 파이썬 세계의 사촌 |
|---|---|---|---|
| 언어 | **TypeScript** | 5.x | 타입 힌트가 **강제되는** 파이썬 |
| 런타임 | 브라우저 + **Node.js** | Node 20 | 브라우저 = 실행 환경, Node = CPython |
| UI 라이브러리 | **React** | 19.1 | Streamlit·Dash의 "선언형 UI"를 정석으로 |
| 프레임워크 | **Next.js (App Router)** | 15.5 | FastAPI + 템플릿 + 빌드 + 배포를 한 몸에 |
| 스타일 | **Tailwind CSS** | 4.x | CSS를 유틸리티 클래스로 (인라인 스타일의 진화형) |
| 렌더링 | **Canvas 2D API** | 표준 | Pillow/matplotlib을 실시간 60fps로 |
| 오디오 | **Web Audio API** | 표준 | numpy로 파형 합성 → 스피커로 |
| 데이터 | **Supabase (PostgREST)** | REST | 당신이 아는 그 PostgreSQL + 자동 REST API |
| 저장 | **localStorage** | 표준 | 브라우저 안의 작은 key-value 스토어 |
| 배포/PWA | **Vercel + Service Worker** | 표준 | 서버리스 배포 + 오프라인 캐시 |
| 빌드 | **Turbopack / ESLint / next/og** | — | webpack 후계자 · 린터 · 이미지 생성 |

> 이 표의 오른쪽 칸("파이썬 세계의 사촌")이 이 코스의 핵심 교육 전략입니다.
> 낯선 것을 통째로 외우지 말고, **아는 것에 걸어(anchor)** 이해하세요.

## 5일 로드맵 (하루 6~8시간 기준, 조절 가능)

각 날은 **"개념 → 실제 코드 정독 → 손으로 실습 → 셀프 체크"** 흐름입니다.
아침엔 이론, 오후엔 코드, 저녁엔 실습 — 이 리듬을 권장합니다.

### [Day 1 — 언어와 런타임: JS/TS와 브라우저의 사고방식](./day-1-언어와-런타임.md)
파이썬과 자바스크립트의 근본적 세계관 차이부터. 타입스크립트를 "강제되는 타입 힌트"로,
이벤트 루프를 "asyncio의 사촌"으로, ESM 모듈을 "import의 방언"으로 흡수합니다.
`package.json`을 `pyproject.toml`에 대응시키며 툴체인을 잡습니다.
**끝나면**: TS 코드를 두려움 없이 읽고, 비동기 코드의 실행 순서를 예측할 수 있다.

### [Day 2 — React 19 & Next.js 15 App Router](./day-2-react-nextjs.md)
"화면을 명령하지 말고 선언하라"는 React의 패러다임 전환. 훅(useState·useEffect·
useRef), 컴포넌트, JSX, 그리고 서버/클라이언트 컴포넌트의 경계. Next.js의
파일 기반 라우팅을 FastAPI 라우터와 비교하며 익힙니다.
**끝나면**: 컴포넌트를 만들고, 상태를 다루고, 리렌더가 언제 일어나는지 설명할 수 있다.

### [Day 3 — 실시간 렌더링: Canvas · 게임 루프 · Web Audio](./day-3-렌더링과-게임루프.md)
이 프로젝트의 심장. "초당 60번 변하는 건 캔버스, 가끔 변하는 건 React"라는
대원칙의 실체. `requestAnimationFrame` 루프, `dt` 기반 프레임 독립성,
update/draw 분리, 그리고 오실레이터로 소리를 합성하는 법.
**끝나면**: 프레임 독립적 애니메이션 루프를 직접 짜고, 성능을 논할 수 있다.

### [Day 4 — 스타일링과 플랫폼: CSS/Tailwind · PWA · 접근성 · 배포](./day-4-스타일과-플랫폼.md)
CSS 박스 모델·flexbox·캐스케이드(파이썬엔 없는 세계). Tailwind 유틸리티 사고법.
반응형·모바일·safe-area. 서비스 워커와 오프라인 캐시 전략. 접근성. 그리고

**끝나면**: 반응형 UI를 스타일링하고, 앱을 배포하고, 소셜 프리뷰를 이해한다.

### [Day 5 — 데이터·프로덕션·1티어로 가는 길](./day-5-데이터와-프로덕션.md)

**끝나면**: 데이터 계층을 설계하고, 실무 코드를 읽고 고치고, 스스로 성장할 지도를 갖는다.

## 시작하기 전에 — 환경 준비 (30분)

```bash
# 1. Node.js 확인 (없으면 https://nodejs.org 에서 LTS 설치 — 파이썬의 python3에 해당)
node --version   # v20 이상 권장
npm --version    # Node에 딸려 옴 (pip에 해당)

# 2. 이 저장소에서 의존성 설치 (pip install -r requirements.txt 에 해당)
npm install

# 3. 개발 서버 실행 (uvicorn main:app --reload 에 해당)
npm run dev
#  → http://localhost:3004 접속. 게임이 뜨면 준비 완료!

# 4. 타입 검사 & 린트 (mypy · ruff 에 해당)
npx tsc --noEmit   # 타입 오류 검사
npm run lint       # 코드 스타일 검사
```

**권장 도구**
- **에디터**: VS Code + 확장 "ESLint", "Tailwind CSS IntelliSense". (당신이 PyCharm/VSCode를
  쓰던 그 방식 그대로. TS는 에디터의 자동완성·타입 추론이 학습을 절반으로 줄여 줍니다.)
- **브라우저**: Chrome — DevTools(F12)가 이 코스 내내 당신의 `pdb`이자 `htop`입니다.

## 학습을 200% 끌어올리는 3가지 습관

1. **파일을 열어 놓고 읽어라.** 이 문서의 `src/lib/mascot.ts:42` 같은 좌표는 장식이
   아닙니다. 실제로 열어서 문서의 설명과 코드를 나란히 두세요. 이 저장소는 그러라고
   주석을 한글로 빼곡히 달아 놨습니다(§13 학습용 저장소 원칙).
2. **고쳐 보고 깨뜨려 보라.** "이 숫자를 2배로 하면?" "이 `try/catch`를 지우면?"
   `npm run dev`는 저장하는 순간 화면을 갱신합니다(핫 리로드). 실험 비용이 0에 가깝습니다.
3. **파이썬으로 번역하라.** 새 개념을 만나면 "파이썬이라면 이게 뭐지?"를 먼저 물으세요.
   대응이 안 되는 지점(예: 브라우저 이벤트 루프의 단일 스레드성)이 바로 **진짜 새로 배울
   것**입니다. 그 지점에 에너지를 집중하세요.
