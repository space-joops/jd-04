## 1. 게임 한 줄 소개 (엘리베이터 피치)

**"버추얼 조이스틱으로 슝슝 추진해서, 하늘에서 떨어지는 우주쓰레기를 몽땅
먹어치우는 모바일 우선 캐주얼 게임."**

- 장르: 원포인터 캐주얼 아케이드 · 서바이벌 스코어 어택
- 플랫폼: 모바일 브라우저 우선, 데스크톱 마우스도 완전 지원
- 플레이 시간: 한 판 30초~3분. "한 판만 더!"를 유도하는 짧은 루프
- 조작: 오직 포인터(터치/마우스) 하나 — 누른 곳이 조이스틱이 되고, 끌어서 추진한다 (§6-1).
  버튼 없음, 튜토리얼 없이 3초 안에 이해 가능할 것
- 아트는 **레트로 픽셀(8비트) 스타일** — 모든 그림을 가상 픽셀 격자에 코드로 찍는다 (§11)

## 2. 세계관 · 테마

- 배경: **2061년, 지구 저궤도(LEO)**. 2031년에 케슬러 신드롬(파편이 파편을 낳는
  연쇄 충돌)이 실제로 일어나, 저궤도는 "8,000톤짜리 쓰레기 구름"이 됐다.
  (현실의 우주쓰레기 문제를 귀엽게 비튼 설정.)
- **자매작 STELLAPET과 세계관 공유**: 인류의 대답은 로켓도 레이저도 아닌,
  **우주쓰레기를 먹고 자라는 생체 위성** — 지상에서 다마고치처럼 키워 궤도로
  올려 보낸다(STELLAPET 프로젝트의 이야기). SPACE JOOPS의 주인공은 그렇게
  올라온 아이들 중 하나다. 랜딩 페이지 "/"가 이 스토리를 전한다 (§4).
- 주인공: **입 큰 민트색 우주 친구(마스코트)**. 궤도 청소부 생체 위성.
  우주쓰레기가 주식(主食)이다.
- 톤 앤 매너: 심각한 환경 문제를 다루지만 분위기는 끝까지 **귀엽고 낙천적**으로.
  죽음/폭발 대신 "배탈", 적 대신 "뾰족뾰족한 애들"이라는 어휘를 쓴다.
- 인게임 카피는 **레트로 아케이드풍 영어** ("YUM!", "GAME OVER", "TAP TO START") —
  픽셀 폰트 Press Start 2P에 한글 글리프가 없기 때문 (§11). 짧고 경쾌하게, 죽음/폭발
  어휘는 여전히 금지. 코드 주석·문서·메타데이터는 계속 한글 구어체.
- **다국어(i18n) — HTML UI만 10개 언어, 캔버스는 영어 고정.** UI 텍스트(메뉴·설정·
  툴팁·버튼·스토리)는 사전(`src/lib/i18n/`)으로 10개 언어(en 기본·ko·ja·zh·es·fr·
  de·pt·ar·sw) 번역한다. 언어는 설정에서 고르며(기본 "auto"=브라우저 언어), 아랍어는
  RTL(`<html dir="rtl">`). **브랜드·HUD 아케이드 간판(SPACE JOOPS·GAME OVER·SCORE·
  TAP TO START·telemetry 라벨 LAT/LON 등)은 영어 고정** — 오락실 카피는 만국 공통이고
  Press Start 2P가 라틴 전용이라 픽셀 톤이 산다. 단, **캐릭터 반응 팝업(YUM!·OUCH!·
  FUEL UP!·COMBO·GEO ORBIT! 등, `fx.*` 키)은 언어 설정을 따라 번역**된다 — 캔버스
  `fillText`가 CJK·아랍은 시스템 폰트로 폴백해 그린다(픽셀 톤은 en/ko만, HTML UI와 같은
  절충). 사전 키의 정본은 `dicts/en.ts`이고, `type Dict = Record<키, string>`이라
  번역이 빠지면 컴파일이 막힌다.

## 3. 핵심 게임 루프

```
쓰레기가 하늘에서 떨어진다
      ↓
조이스틱으로 추진해 받아먹는다 (+점수, 몸이 커짐 — 연료를 아껴 쓰며!)
      ↓
가시(hazard)를 피한다 — 닿으면 하트 -1
      ↓
점수가 오를수록 낙하가 빨라지고 가시가 늘어난다 (난이도 상승)
      ↓
하트 3개를 다 잃으면 게임 오버 → 최고 기록 갱신 → 즉시 재도전
```

## 4. 화면 흐름 (라우트 & 상태 기계)

| 라우트 | 역할 |
|---|---|
| `/` | 랜딩 페이지: 타이틀 + 규칙 + TAP TO START(`/play?start=1`) + 최고 기록(`BestScore`) + 앱 설치·공유 버튼. 세계관 스토리(§2)는 **첫 방문에만 전체화면 스타워즈 크롤 인트로**(`StoryIntro`)로 1회 재생 — 20초, 탭/ESC로 스킵, 끝나면 자동 닫힘, 본 여부는 `localStorage`(키: `sjs-intro`). **크롤 텍스트 뒤에는 이야기를 설명하는 삽화 몽타주**(`StoryScene`) — 3막(케슬러 파편 증식→생체 위성 발사→궤도 청소)을 기존 그리기 함수로. 이후 방문은 푸터의 "스토리 다시보기"로만 연다. prefers-reduced-motion이면 크롤·삽화 대신 정지 화면. 배경에서 **어트랙트 모드**(`AttractSky`) — 마스코트가 혼자 떨어지는 쓰레기를 받아먹는 오락실 대기 화면(가시 없음, 60% 속도). **배경 달을 탭하면 게임과 같은 음력 이스터에그 토스트**(§11). |
| `/rank` | 랭킹 전용 페이지: 단판·누적 **TOP 10** 두 순위표 (1~3위 메달, 내 펫 ★ 강조), TAP TO START·HOME 링크. 랜딩의 "🏆 랭킹 보기"로 진입. 기록 제출 없음 — 순수 조회. |
| `/bag` | 인벤토리(로컬 도감, §8-2): 종류별 수집 개수 + 게임과 같은 `drawJunk` 스프라이트를 미니 캔버스로 렌더한 카드 그리드. 못 먹은 종류는 흐린 실루엣 + "???" (도감의 재미). 랜딩의 "🎒 인벤토리"로 진입. |
| `/orbit` | 궤도 모니터(§8-3): 내 펫(생체 위성)이 지금 지구 어디 위를 나는지 실시간 telemetry. 펫이 없으면 안내 + TAP TO START만. 랜딩의 "🛰️ 궤도 모니터"로 진입. |
| `/settings` | 설정(§8-4): 캐릭터 3종 선택·기지국 위치(위/경도)·시간 표시 형식. 변경 즉시 `localStorage`(키: `sjs-settings`)에 저장. 랜딩의 "⚙️ 설정"으로 진입. |
| `/play` | 게임 본편. **등록된 펫이 없으면 이름 등록 게이트(`PetNameGate`, §8-1)가 먼저 뜬다.** 내부 상태 기계 `Phase = "title" → "playing" → "over"`. **`?start=1`로 진입하면(랜딩의 TAP TO START) 타이틀을 건너뛰고 즉시 시작** — 두 번 탭하게 하지 않는다(게이트 통과 직후에도 동일). 직접 방문은 평소처럼 타이틀 데모. 오디오는 게임 중 첫 터치가 깨운다(§12). HUD 오른쪽 **하트 옆 HOME 버튼**으로 언제든 랜딩("/")으로 복귀 — 이 링크만 `pointer-events-auto`이며 HUD 침범 금지선 안이라 조이스틱과 안 겹친다. |

**Phase 규칙:**
- `title`: 게임 방법 안내. 배경에서 쓰레기가 60% 속도로 느긋하게 떨어지는 데모.
  **데모에는 가시가 스폰되지 않는다** (구경 중 주인공이 찔리는 연출 방지). 탭하면 시작.
- `playing`: 본 게임. 충돌 판정은 이 phase에서만 활성화.
  **일시정지**: HUD ⏸ 버튼 또는 백그라운드 전환(visibilitychange — 전화·알림에
  억울하게 죽지 않게 자동 정지, 복귀 시 자동 재개는 안 한다: 준비된 건 화면이지
  사람이 아니다). update만 멈추고 draw는 유지, 멈출 때 조이스틱을 놓는다(재개
  폭주 방지). 탭 또는 ▶로 재개.
- `over`: 결과 화면 (점수, 먹은 개수, 최고 기록, 신기록 축하).
  **게임오버 후 0.6초간 재시작 입력 무시** — 죽는 순간 누르고 있던 손가락 때문에
  결과를 읽기도 전에 재시작되는 사고 방지 (입력 디바운스). 탭하면 재도전.

단, 쓰레기를 놓치면 콤보는 끊긴다 (§5-1).

### 5-1. 콤보 배율

**"안 놓치고 연속으로 먹으면 점수가 뻥튀기된다"** — 실력 보상 루프 (§16에서 승격).

- 쓰레기 4종을 먹을 때마다 콤보 +1. **5개마다 배율이 ×2, ×3… 최대 ×5** (`comboStep`, `comboMaxMult`).
- 점수 = 10 × 배율. **별 점수도 배율이 실린다** (40 × 배율 — 콤보 중의 별은 잭팟).
  별이 하트를 회복할 때는 배율 무관.
- 콤보가 끊기는 때: **쓰레기를 놓쳤을 때**(화면 아래로) / **가시에 맞았을 때**.
- 별·연료는 콤보를 **올리지도 끊지도 않는다** — 보너스를 놓쳤다고 벌주면 억울하다 (§7의 정신).
- 피드백 (§10): 배율이 오르는 순간 "COMBO x2!" 팝업(노랑) + 옥타브 도약 2음 팡파레.
  HUD 점수 아래 `x2 COMBO` 상시 표시 — 단, ×2부터만 (×1은 정보가 아니라 소음).

### 5-2. 파워업 3종

한시적 버프. 스폰 4% 고정을 셋이 균등하게 나눈다 (§9). 콤보를 올리지도
끊지도 않고, 놓쳐도 콤보 무사 (별·연료와 같은 취급).

- **자석 강화 (magnet, 8초)**: 흡입 범위·당기는 힘 ×3. 평소 자석(§7)은 몰래
  돕지만 파워업만은 **범위 링을 그려서 티를 낸다** — "지금 강하다"의 전달.
- **시간 느려짐 (slowmo, 8초)**: 낙하물의 시간만 ×0.45로 — 주인공은 평소
  속도라 상대적으로 강해진 기분. 화면에 라벤더 기운, 끝나기 1초 전부터 옅어진다.
- **방패 (shield, 1회)**: 시간제가 아니라 **다음 가시 한 방을 대신 맞는다** —
  하트도 콤보도 무사 (콤보 지킴이). 몸 주위를 도는 민트 도트 6개로 표시,
  막는 순간 "BLOCKED!" + 안도의 2음 + 짧은 진동(30ms) + 잠깐의 무적(0.8초).
- 획득 피드백: 이름 팝업("MAGNET!" 등, 파워업 색) + 충전 스윕 사운드.
  자석·슬로모는 연료 게이지 아래 **남은 시간 미니바** (버프 색 그대로).

## 6. 주인공 (마스코트) 요구사항

### 6-1. 이동 — 버추얼 조이스틱 + 추진 물리
- **누른 곳이 조이스틱 원점**, 끌면 그 방향으로 추진(가속). 고정 조이스틱이 없어
  화면이 깨끗하고, 어디를 눌러도 되니 설명이 필요 없다. 놓으면 관성으로 미끄러진다.
- 기울기(끈 거리, 최대 60px)를 **3단계로 양자화**: <25px 약 / <45px 중 / 그 이상 강.
  단계별 가속량 `thrustSpeeds: [100, 250, 450]`px/s². 5px 미만은 데드존(손떨림 방지).
- **연료 시스템**: 최대 3000. 단계별 초당 소모 `[10, 40, 100]` — 세게 밀수록 기하급수로
  비싸다. 조이스틱을 놓고 있으면 초당 600 회복 — "쉬어 가는" 리듬을 만드는 장치.
  연료(배터리) 아이템을 먹으면 +800. 바닥나면 추진 불가(관성만 남는다).
- **우주 관성**: 속도는 지수 감쇠 마찰(`friction: 1.2`)로 천천히 줄고, 최소 유지 속도
  30px/s 아래로는 떨어지지 않는다 — 완전히 멈추지 않고 늘 둥둥 떠다니는 부유감.
- 화면 경계·상단 HUD 영역(y < r+64)에 닿으면 **반발 계수 0.8로 튕긴다**.
- title/over 화면에서는 조작 불가, 화면 중앙 아래에서 sin 곡선으로 둥둥 떠다닌다.

### 6-2. 성장 시스템 (핵심 보상 루프)
- 시작 반지름 24px → 하나 먹을 때마다 +0.45px → **최대 38px 상한**.
- 커질수록 판정도 커져서 더 잘 먹게 되는 **선순환**. 상한이 있어 무한히 쉬워지진 않음.
- 가시에 맞으면 -3px (최소 24px). "아프면 살짝 작아진다"는 시각적 페널티.

### 6-3. 겉모습 · 생명력 연출
- 8×8 가상 픽셀 격자에 찍은 **민트색 픽셀 슬라임** + 도트 눈 2개 + 입 + 볼터치 +
  노란 불빛 안테나 (§11). 그리기는 `src/lib/mascot.ts`에 격리.
- **캐릭터 3종**(§8-4, 설정에서 선택): 민초·딸기·라벤더 — `drawMascot`의 `variant`가
  몸통 색뿐 아니라 안테나·눈·입 모양까지 바꾼다. 색·이름은 `constants.ts`의
  `MASCOT_VARIANTS`가 원본. 설정이 없으면 민초(기본).
- **살아있는 표정** — "무엇을 보여줄지"는 게임 본체 update가 계산해 `MascotFace`로
  넘기고, mascot.ts는 그리기만 한다 (update/draw 분리 §12). 모든 phase에서 돈다
  (타이틀 데모의 마스코트도 살아 있어야 한다):
  - **시선**: 가장 가까운 먹이 방향을 본다. 눈동자는 지수 감쇠로 부드럽게 따라가되,
    그릴 때는 픽셀 단위(-1/0/+1칸)로 반올림 — 소수점 이동은 도트 격자를 깨뜨린다.
  - **깜빡임**: 2.2~4.7초 랜덤 간격으로 0.13초간 눈꺼풀 선. 간격이 랜덤이어야
    로봇 같지 않다. (무적 중 반투명 깜빡임 §8과는 별개.)
  - **입벌림**: 먹이가 사정거리(r+120px) 안에 오면 입이 2×2 → 4×2.5로 활짝.
    중간 단계 없이 두 상태만 — 픽셀 아트는 단계가 분명해야 산다.
    벌린 만큼 **획득 판정 +10px 보정** (연출과 판정의 일치, §7).

## 7. 판정 규칙 — 🎮 황금률

> **피격 판정은 보이는 그림보다 작게, 획득 판정은 크게.**
> 플레이어가 "억울하게 죽었다"고 느끼면 안 되고, "아슬아슬하게 먹었다"고 느껴야 한다.

- 가시 피격: `거리 < 주인공r + 가시size × 0.75` (그림은 가시 끝까지 size×1.0 — 판정은 그보다 작게)
- 먹이 획득: `거리 < 주인공r + 먹이size × 0.65 + 입벌림 × 10px`
  — 입을 벌린 만큼 판정도 커진다. 벌린 입으로 들어가는 게 보이면 판정도
  그래야 억울하지 않다 (연출과 판정의 일치, §6-3).
- **자석 효과**: 먹이가 주인공 반경 `r + 70px` 안에 들어오면 슬쩍 끌려온다.
  플레이어가 눈치채지 못할 강도로 — 폰에서 픽셀 단위 조준은 불가능하니 게임이 몰래 돕는다.
  (⚠️ 끌어당길 때는 `x`가 아니라 흔들림 중심축 `x0`를 옮겨야 한다. `x`는 매 프레임 재계산되는 파생값.)
- 먹기 시작하면(`eatT ≥ 0`) 0.16초 동안 입으로 빨려 들어가며 축소·고속회전 후 소멸("꿀꺽").

## 8. 목숨 · 무적 · 게임 오버

- 시작 하트 3개. 가시에 닿으면 -1. 0이 되면 게임 오버.
- 피격 직후 **1.4초 무적** — 없으면 가시 하나에 하트가 연달아 증발한다. 필수 안전장치.
- 무적 중에는 주인공이 초당 8회 반투명 깜빡임 (무적 상태의 시각적 전달).
- 게임 오버 시 최고 기록이면 `localStorage`(키: `sjs-best`)에 저장.

### 8-1. 온라인 리더보드 (Supabase) — 펫 이름 · 단판/누적 이원 경쟁

**펫 등록**: 첫 플레이 진입 시 펫 이름(1~10자, 한글 환영)을 한 번 짓는다
(`pet-name.tsx` — 이름이 없으면 게임을 마운트하지 않는 게이트). 이때 만든
uuid(`crypto.randomUUID`)와 이름이 `localStorage`(키: `sjs-pet`)에 저장되고,
이후 모든 기록은 이 펫으로 자동 제출된다. **경쟁 키는 이름이 아니라 uuid**,
그리고 **이름은 유일하다** (대소문자 무시 유니크) — 이름 입력 순간
`register_pet` RPC가 서버에 이름을 **선점**하고, 중복이면 409를 받아
"다른 이름을 지어줘" 안내를 띄운다. 게임오버 때가 아니라 짓는 순간 알려주는
게 핵심 — 한 판을 다 놀고 나서 이름이 겹친다는 걸 알면 억울하다.
개명은 없다(이름은 최초 등록 때 고정). 네트워크 오류로 선점을 못 했으면
게임은 그냥 시작시키고(§12) 게임오버 제출 때 자동 재등록한다(자기 치유).
localStorage가 지워지면 새 펫으로 다시 시작한다 (오락실 문법).

**두 개의 경쟁** (게임오버 화면에서 탭 전환):
- **단판 TOP 5** (`pets.best_score`): **펫당 한 줄** — 자기 최고 기록을 깰 때만
  갱신되므로 같은 펫이 1·2등을 동시에 차지할 수 없다. 동점은 먼저 세운
  쪽(`best_at`)이 위. 판마다의 원본 기록은 `scores`에 히스토리로 남는다
  (랭킹엔 안 쓰고 미래 기능의 재료).
- **누적 TOP 5** (`pets.total_eaten`): 등록된 펫이 **통산 수거한 쓰레기 개수**
  랭킹. 연료·별은 eaten에 안 세므로 순수 쓰레기만 집계된다. 내 펫은 ★로 강조.

**자동 제출**: 게임오버가 되면 이름 입력 없이 기록이 자동 전송된다(0점 제외).
제출 완료를 기다렸다가 순위를 조회해 방금 판이 바로 반영돼 보인다.

- 기술: Supabase PostgREST를 `fetch`로 직접 호출 (`src/lib/leaderboard.ts`) —
  조회 2개(단판·누적) + RPC 2개(`register_pet`·`submit_result`). SDK를 쓰지
  않는 이유: 의존성보다 HTTP가 그대로 보이는 쪽이 학습용 저장소(§13)에 맞다.
- **쓰기는 RPC 두 개로만**: `register_pet`이 이름을 선점하고(중복이면 409),
  `submit_result`는 **등록된 펫만** 받아 누적 가산과 단판 기록 insert를
  원자적으로 처리한다(security definer). 제출 경로에 이름 파라미터가 없어
  개명·무단 등록이 샐 구멍이 없다. anon 키로는 테이블 직접 INSERT/UPDATE
  불가 — 남의 누적치를 덮어쓸 방법이 없다. 함수 안 검증(이름 1~10자,
  점수 1~100000, eaten 0~10000)이 쓰레기 값을 거른다. 익명 제출 자체의
  조작은 완전히 못 막는다 — 아케이드 리더보드의 전통적 타협.
- 설정: `.env.local`에 `NEXT_PUBLIC_SUPABASE_URL` / `NEXT_PUBLIC_SUPABASE_ANON_KEY`
  (`.env.example` 참고), `supabase/schema.sql`을 SQL Editor에서 실행 (v1에서
  재실행하면 안전하게 마이그레이션된다).
- **철저한 실패 허용 (§12)**: env 미설정이면 리더보드 UI가 통째로 숨는다 —
  클론만 받아도 게임은 로컬 최고 기록만으로 완전 동작. 네트워크·서버 오류도
  조용히 생략하고 게임을 절대 막지 않는다.
- UI 규칙: 게임오버 오버레이는 `pointer-events-none`이 원칙이지만(§12),
  단판/누적 탭 버튼만 `pointer-events-auto` — 버튼을 누른 게 "탭해서
  재시작"으로 새지 않게 하는 장치.

### 8-2. 인벤토리 (로컬 도감)

**먹은 아이템을 종류별로 세는 내 가방** — 서버가 아니라 이 기기의
`localStorage`(키: `sjs-inventory`)에만 쌓인다 (리더보드 §8-1과 무관한 순수 로컬 재미).

- 기록 시점: `eat()`에서 **먹는 순간마다 즉시 +1·저장** — 게임 도중 탭이 닫혀도
  그때까지의 수집은 남는다. 가시는 "먹는" 게 아니므로 기록되지 않는다.
- 통산 누적: 판이 끝나도, 며칠이 지나도 계속 쌓인다 (오락실 도감 문법).
- 보기: `/bag` 페이지 (§4). 카드 그림은 이미지가 아니라 게임의 `drawJunk`를
  미니 캔버스에 그린 것 — 게임 도트가 바뀌면 도감도 저절로 같아진다 (§11).
- 종류별 한글 이름은 `constants.ts`의 `JUNK_NAMES`가 원본.
- 못 먹은 종류: 흐린 실루엣 + 이름 회색 + 설명 "???" — 다 모으고 싶게.

### 8-3. 궤도 모니터 (실시간 궤도역학)

**내 펫(생체 위성, §2)이 지금 지구 어디 위를 날고 있는지 보여주는 화면**
(`/orbit`). 서버·로컬 어디에도 펫의 "위치"라는 데이터는 없다 — 대신 펫의
uuid를 씨앗 삼아 궤도 요소(고도·경사각·승교점 경도·발사 시점)를
**결정론적으로** 생성하고, **실제 케플러 궤도역학(원궤도 근사)**으로 진짜
현재 시각 기준 위치를 계산한다(`src/lib/orbit.ts`). 같은 펫은 언제 다시
봐도 같은 궤도 — 그 위에서 "지금 이 순간"의 위치만 계속 달라진다.

- 궤도 요소: 고도 400~500km(LEO), 경사각 30~98°, 승교점 경도 -180~180°,
  발사 시점은 2015~2023년 사이(펫마다 다름) — 항상 실제 현재 시각보다
  과거여야 REV(공전 횟수)가 음수로 뒤집히지 않는다.
- 물리: 케플러 제3법칙으로 공전 주기, 원궤도 지상궤적 공식으로 위/경도(지구
  자전 반영 — 매 주기 서쪽으로 이동), 원궤도 속력 `v = √(μ/a)`. J2 섭동 등은
  다루지 않는 원궤도 근사 — "진짜(약간) 궤도 시뮬레이션" 정도의 재미가
  목적이지 정밀 항법이 아니다.
- 화면 구성(전부 픽셀 격자 문법 §11으로 재해석):
  - **씬 캔버스 + 줌 3단계**(`src/lib/globe.ts`·`solar.ts`): 줌 인=근접 지구본,
    한 단계 아웃=지구+**진짜 공전하는 달**(배경의 잠자는 가짜 달은 이때 숨김 —
    `drawBackdrop`의 `decorMoon=false`), 두 단계 아웃=**태양계 전체**(태양+수성~
    해왕성, 지구에 "YOU ARE HERE"). 달·행성은 실제 공전주기 비율로 돌고 시간
    가속에 반응한다. 거리는 축척이 아니라 등간격(바깥 행성까지 화면에 담기게).
  - **지구본 카메라는 펫을 따라간다**(`spinDeg = -경도`) — 늘 정면 중앙에 두어
    "지구본에서 줍스가 어디 있는지" 한눈에 보이고, 펫이 움직이면 그 아래로
    대륙이 흐른다. 펫 마커는 네모 점이 아니라 **게임과 똑같은 `drawMascot`**(작게)
    + 노란 표적 도트로 그린다.
  - **2D 세계지도**(`src/lib/worldmap.ts`, equirectangular): 궤도 전체의 지상궤적
    (S자)과 펫 위치를 평면 지도에 보여준다. 대륙은 GeoJSON 없이 몇 개의 "대륙
    얼룩(blob)"으로 픽셀 격자를 칠한 근사(지리 정확도 아님, §11). 주야 경계(밤
    반구)를 어둡게 덮는다.
  - **telemetry 카드**(LAT·LON·ALT·VEL·PERIOD·REV) — 각 항목을 탭/호버하면 쉬운
    한글 설명 툴팁(ⓘ). SUNLIT/ECLIPSE 배지.
  - **가상 시계 + 시간 가속**(1× / 60× / 1h/s / 1d/s): 빠르게 돌리면 지상궤적·
    달·행성이 눈에 보이게 움직여 이동 경로를 확인할 수 있다. LOCAL(펫 지상
    지점의 현지 태양시)·UTC 표시. 1×는 실제 시각에 재동기화, 가속은 자체 전진.
  - **"궤도역학이 뭐야?" 설명 모달**(`orbit-explainer.tsx`): 우리 톤의 쉽고
    재미난 3컷 설명(왜 안 떨어지나·높으면 느긋·지구가 도니 서쪽으로 밀림) +
    미니 캔버스 픽셀 그림(drawMascot·픽셀 지구 재사용).
- telemetry·시계 숫자는 초당 4회로 스로틀 갱신(§12 — 60fps로 React 리렌더 금지).
- **기본 시간 배속은 60×** — 진입하자마자 궤도가 도는 게 보이게(1×는 버튼으로).
- 시계 두 번째 칸은 설정(§8-4)의 시간 형식을 따른다: UTC / 기기 현지 / 기지국 태양시.
- 펫이 없으면(첫 방문) 안내 문구 + `/play?start=1` 링크만 — 등록 게이트는
  중복 구현하지 않고 `/play` 전용으로 유지한다.

### 8-4. 설정 (`/settings`)

**캐릭터·위치·시간 형식을 고르는 화면.** 서버가 아니라 이 기기의
`localStorage`(키: `sjs-settings`)에만 저장된다(리더보드와 무관). 변경 즉시
저장 — "저장 버튼"을 따로 두지 않는다(오락실 문법).

- **캐릭터 3종**(`MASCOT_VARIANTS`, `constants.ts`가 원본): 민초(민트)·딸기(분홍)·
  라벤더 — 색뿐 아니라 **안테나·눈·입 모양**까지 다르다(`drawMascot`의 `variant`
  분기, §6-3). 고른 캐릭터는 게임 마스코트·궤도 지구본/세계지도 마커·설정 미리보기에
  모두 반영된다. **앱 아이콘(`pwa-icon.tsx`)은 브랜드 고정이라 민초 그대로.**
- **언어**(§2 i18n): 10개 언어 중 선택 + "자동"(브라우저 언어). 바꾸면 화면이 즉시
  그 언어로(아랍어는 RTL). 저장은 `sjs-settings.language`, 실제 적용은 `I18nProvider`.
- **기지국 위치**(위/경도): 네 가지로 고른다 — `navigator.geolocation` "현재 위치"
  버튼 / 수동 위경도 입력 / **2D 세계지도 탭해서 찍기**(`location-picker.tsx`,
  `worldmap.ts`의 `drawLocatorMap` 재사용) / **도시 검색**(`cities.ts`의 주요 도시
  목록). 달 위상의 반구(§11)와 궤도 모니터의 "기지국 태양시"(§8-3)에 쓰인다.
- **시간 표시 형식**: UTC / 기기 현지 / 기지국 태양시(위치 필요). 각 형식의 **뜻 설명**을
  아래에 한 줄로 보여준다(기지국 태양시 = 경도 15°=1시간으로 UTC를 민 평균 태양시).
  궤도 시계에 반영.
- 전부 실패 허용(§12) — env·권한 없이도, 설정이 없어도 기본값(자동 언어·민초·UTC)으로 완전 동작.

## 9. 스폰 · 난이도 곡선

난이도는 시간이 아니라 **점수**로 오른다 (잘할수록 어려워지는 실력 비례 설계).

- 난이도 계수: `difficulty = min(1, score / 400)` — 400점에서 최고 난이도 도달.
- 스폰 종류 확률 (추첨 순서대로): **별 7% 고정** (난이도 무관 — 언제나 같은 희망)
  → **파워업 4% 고정** (3종 균등, §5-2) → **연료 6% 고정** (쓰레기 종류가 늘어도
  연료 경제가 흔들리지 않게 별도 추첨) → 가시 10%에서 난이도에 따라 **최대 23%**
  → 나머지는 쓰레기 8종 균등.
  별·파워업·연료는 성장 없음: 몸은 쓰레기를 먹어야만 커진다.
- 낙하 속도: 기본 55~100px/s × `(1 + difficulty × 1.1)` — 최대 2.1배.
- 스폰 간격: `max(0.42, 1.05 - score/900)`초 — **바닥 0.42초** (무한히 어려워지지 않게).
- 스폰 간격에 **±30% 랜덤 지터** 필수 — 정확히 일정한 리듬은 외워져서 지루해진다. 불규칙성도 설계다.
- 타이틀 데모: 가시 없음, 낙하 속도 60%, 간격 1.5초 고정.

## 10. 게임 필(juice) 요구사항

모든 사건에 **시각 + 소리 + 진동** 3중 피드백이 동시에 터진다.

| 사건 | 시각 | 소리 | 진동 |
|---|---|---|---|
| 먹이 먹음 | "YUM!"류 랜덤 팝업(흰색) + 몸 성장 + **먹이색 스파크 7개** | 430→900Hz 상승 트라이앵글 | 12ms |
| 연료 획득 | "FUEL UP!" 팝업(민트) + 연료 게이지 회복 + 스파크 | 먹이와 동일 (상승 트라이앵글) | 12ms |
| 별 획득 | "+40!"(노랑) 또는 "+♥"(분홍) 팝업 + 노랑 스파크 | 660→880→1320Hz 상승 아르페지오 — "더 좋은 일" | 12ms |
| 가시 피격 | "OUCH!"류 랜덤 팝업 + **화면 흔들림 0.35초** + 무적 깜빡임 + **빨강 스파크 10개** | 220→65Hz 하강 톱니파 | 90ms |
| 콤보 배율 상승 | "COMBO xN!" 팝업(노랑) | 옥타브 도약 2음 (523→1046Hz) | — |
| 파워업 획득 | 이름 팝업(파워업 색) + 버프 시각 효과(§5-2) | 330→1320Hz 충전 스윕 | 12ms |
| 방패로 막음 | "BLOCKED!" + 민트 스파크 7개 | 262→392Hz 안도의 2음 | 30ms |
| 고도 상승 | "GEO ORBIT!"/"MOON ZONE!" 팝업 + 배경 교체(§11) | 별과 동일 아르페지오 | — |
| 게임 오버 | 결과 화면 (반투명 검정 오버레이) | 392→330→262Hz 하강 3음 | — |

- **추진 피드백**: 추진 중에는 이동 반대쪽으로 분사 불꽃(단계별 길이·색: 하늘→노랑→빨강,
  매 프레임 ±30% 떨림), 조이스틱 손잡이도 같은 단계 색 — 연료 소모를 몸으로 느끼게.
- **연료 게이지**: 상단 중앙 바. 잔량 20% 이하면 네온 민트 → 빨강 경고.

- 사운드 문법: **음이 올라가면 긍정, 내려가면 부정. 부드러운 파형은 좋은 일, 거친 파형(sawtooth)은 나쁜 일.**
- 팝업 글자: 0.85초 수명, 위로 34px/s 떠오름, 수명의 70%까진 완전 불투명 후 페이드아웃
  ("충분히 보여준 다음 사라진다"). 배경색 두꺼운 테두리 + 채움 (어떤 배경에서도 가독).
- 화면 흔들림: 개별 오브젝트가 아니라 **좌표계 전체**를 흔들고, 남은 시간에 비례해 잦아든다.
- 스파크 입자: 2~4px 정수 크기의 **네모 도트**(원은 픽셀 문법 위반 §11).
  각도는 균등 분할 + 지터(완전 랜덤은 뭉치고 완전 균등은 기계 같다), 우주라
  중력 대신 지수 감쇠로 흩어지고, 수명 마지막 40%에서 페이드아웃.
- 소리·진동은 미지원 환경에서 **조용히 생략** — 게임 진행을 절대 막지 않는다.
  (진동은 안드로이드 크롬 등 `navigator.vibrate` 지원 기기에서만.)
- **소리 토글**: HUD의 🔊/🔇 버튼. 설정은 `localStorage`(키: `sjs-muted`)에
  기억 — 한 번 끄면 다음 방문에도 꺼져 있어야 한다.

## 11. 아트 디렉션 — "레트로 픽셀, 우주 팔레트"

8비트 아케이드 감성의 **픽셀 아트**. 도형을 부드러운 곡선(arc/roundRect)이 아니라
가상 픽셀 격자에 fillRect로 "찍어서" 만든다.

- 에셋 파일 0개 — **모든 그림은 코드로** (Canvas 2D).
- **가상 픽셀 격자 방식**: 도형을 -4~+4 범위의 픽셀 좌표로 찍고 `ctx.scale(크기/4)`로
  확대한다. 크기가 달라도 도트 실루엣이 유지된다 (마스코트는 `r/4`, 낙하물은 `s/4`).
- 그리기 함수는 전부 `src/lib/` 모듈로 격리 (`mascot.ts`, `debris.ts`, `backdrop.ts`) —
  아트 스타일을 갈아탈 때 이 파일들만 교체하면 되는 구조를 유지할 것.
- 배경: 우주색 `#141838` + 48px 모눈 격자(알파 0.045) + **반짝이는 별 46개**
  (네모 점/＋ 십자 두 종, 2px/4px 두 단계) + 오른쪽 위 **잠자는 달**("z z" 잠꼬대).
  별의 위치·크기·모양은 결정론적 해시로 고정하고 **밝기만** 시간(t)에 따라 숨쉰다 —
  별마다 속도·위상이 달라야 신호등처럼 같이 깜빡이지 않는다.
- **잠자는 달은 진짜 위상**(`moon.ts`): 날짜로 신월→보름→그믐을 계산해 5단계 픽셀
  버킷으로 그리고, 설정(§8-4)의 위치(반구)에 따라 밝은 쪽을 뒤집는다(남반구는 반대).
  게임 화면에선 달을 HUD 하트 아래로 내려 안 겹치게 두고, **탭하면 "음력 약 N일 ·
  위상이름" 토스트**를 띄우는 이스터에그(타이틀·게임오버·일시정지 때만 — 플레이 중
  추진 조작과 충돌 방지). **랜딩 배경 달도 같은 이스터에그**(`AttractSky`가 window
  클릭을 받아 달 히트박스 검사). 음력은 달 나이 근사라 "약"으로 표기한다.
- **점수 고도 연출**: 0~199점 저궤도(아래에 지구 지평선 — 바다·대륙·대기광),
  200점 정지궤도(왼쪽 아래 작아진 지구 원반), 500점 달 근처(크레이터 달 표면,
  하늘의 잠자는 달은 숨김 — 우리가 그 옆에 도착했으니까). 진입 순간
  "GEO ORBIT!"/"MOON ZONE!" 팝업. 재시작하면 다시 저궤도부터. 랜딩 어트랙트는 저궤도 고정.
- 폰트: 픽셀 폰트 **Press Start 2P**, 캔버스와 HTML 모두. 제목·게임오버 등 큰 글자에는
  `textShadow: 4px 4px 0 #000` 하드 섀도로 아케이드 간판 느낌.
  ⚠️ 한글 글리프가 없으므로 게임 안에 보이는 카피는 전부 영어 (§2).
- 한글 픽셀 폰트 **Galmuri11** (`--font-pixel-ko`): 랜딩 페이지의 한글 스토리 전용.
  jsdelivr CDN에서 불러오며 실패 시 폴백 — 부가 기능이 게임을 죽이면 안 된다 (§12).
- HTML 쪽은 `image-rendering: pixelated`로 이미지 보간 금지.
- 팔레트: 마스코트 민트 `#7ee8b2` / 포인트 노랑 `#ffd166` / 위험 빨강 `#ff8080` / 하트 분홍 `#ff8fab`.
  색은 전부 `constants.ts`에서만 정의한다.
- 부드러운 원을 쓰는 예외: 조이스틱 UI·분사 불꽃 — 게임 오브젝트가 아니라 입력/이펙트
  레이어라 도트 문법을 강제하지 않는다.

## 12. 기술 아키텍처 요구사항

- 스택: Next.js 15 (App Router) · React 19 · TypeScript · Tailwind CSS 4 · Canvas 2D · Web Audio ·
  Supabase (REST 직접 호출, §8-1). 외부 게임 라이브러리·이미지·오디오 파일 **사용 금지** (전부 코드로 생성).
- **대원칙: "초당 60번 변하는 것은 캔버스에, 가끔 변하는 것만 React에."**
  게임 상태는 useEffect 클로저 안의 지역 변수. React `useState`는 `ui` 객체 하나뿐이며
  점수/하트/phase가 실제로 바뀔 때만 `pushUi()`로 알린다. 매 프레임 setState 금지.
- **update/draw 분리**: `update(dt)`는 상태만 바꾸고 캔버스를 건드리지 않는다.
  `draw()`는 읽기만 하고 상태를 바꾸지 않는다.
- 모든 움직임은 `값 × dt` (프레임레이트 독립 — 60Hz든 120Hz든 같은 속도).
  `dt`는 0.05초로 상한 (백그라운드 탭 복귀 시 순간이동/터널링 방지).
- 부드러운 추적은 전부 지수 감쇠 패턴: `pos += (target - pos) × min(1, dt × k)`.
- 배열 순회 중 삭제는 반드시 **역순 순회 + splice**.
- 엔티티는 클래스/상속 없이 "객체 배열 + 순수 함수(make/step/draw)" 구조를 유지한다.
- 캔버스: DPR 대응 (상한 2 — 화질 대비 비용 타협), 리사이즈 시 `fitCanvas` 재호출.
- HTML 오버레이(HUD/타이틀/게임오버)는 `pointer-events-none`으로 터치를 통과시킨다 (링크만 auto).
- 캔버스에 `touch-none` 필수 (없으면 드래그를 브라우저가 스크롤로 가로챔). 핀치 줌 금지(viewport).
- 오디오는 반드시 사용자 제스처 핸들러 안에서 `ensure()` (브라우저 자동재생 정책).
- `localStorage`·오디오는 전부 실패 허용(try-catch) — 부가 기능이 게임을 죽이면 안 된다.
- useEffect 정리 함수에서 rAF·이벤트 리스너·AudioContext를 반드시 해제.
- `globalAlpha`는 대입이 아니라 **곱해서** 쓴다 (중첩 투명도 버그 방지).

## 13. 비기능 요구사항

- 성능: 중급 폰에서 60fps 유지. 화면 밖 오브젝트 즉시 제거로 배열 누적 방지.
- 모바일 UX: 노치 대응(safe-area-inset), 길게 누르기/우클릭 메뉴 차단, 텍스트 선택 방지,
  상단 바 색까지 우주색(`themeColor`) — 몰입감.
- **PWA 설치(홈 화면 추가)**: 웹 앱 매니페스트(`manifest.ts` → /manifest.webmanifest) +
  앱 아이콘은 **코드로 생성**(`pwa-icon.tsx`, ImageResponse — 에셋 0개 원칙 §11.
  mascot.ts의 도트 좌표를 div로 옮긴 것이라 도트를 바꾸면 함께 갱신).
  설치 후 전체화면(fullscreen → standalone 폴백)·세로 고정. 랜딩의 **"앱 설치"
  버튼**: `beforeinstallprompt`가 있으면 바로 설치 프롬프트, 없으면(애플처럼
  바로 안 되는 환경) 클릭 시 환경별 안내 패널 — iOS 사파리는 "공유 → 홈 화면에
  추가" 안내, iOS 인앱 브라우저(카카오톡 등)는 **"사파리로 열기"**(x-safari-
  스킴, iOS 17+) + 링크 복사 폴백, 그 외는 브라우저 메뉴 안내.
  이미 standalone 실행 중이면 통째로 숨김.
- **오프라인 구동 & 배포 갱신** (`public/sw.js` + `sw-register.tsx`):
  - 캐시 전략은 요청 종류별로 다르다 — 페이지 이동은 **네트워크 우선**(새 배포가
    캐시에 막히면 안 된다, 오프라인일 때만 셸 폴백), `/_next/static/`·아이콘은
    **캐시 우선**(해시 파일명 = 불변), 그 외 동일 출처 GET은 stale-while-revalidate.
    교차 출처(폰트 CDN·Supabase)와 GET 아닌 요청은 손대지 않는다.
  - 게임 상태는 localStorage(§8)라 오프라인에서도 플레이·저장이 전부 동작.
  - **등록은 프로덕션 빌드에서만** — 개발 중엔 핫리로드가 캐시에 오염된다.
  - **갱신 트리거 = package.json 버전 올리기**: 등록 URL이 `/sw.js?v={버전}`이라
    버전을 올려 배포하면 새 워커가 설치되고, **"새 버전 도착" 토스트** →
    사용자가 누르면 SKIP_WAITING → 교대 → 리로드. 리로드는 사용자가 눌렀을
    때만 한다 — 게임 도중 멋대로 새로고침하면 안 된다.
  - `sw.js`의 `VERSION` 상수는 캐시 저장소 이름 — 캐시 구조를 바꾸거나
    대청소가 필요할 때만 올린다 (activate에서 옛 캐시 전부 삭제).
- **버전 표기**: `package.json`의 version을 빌드 타임에 읽어 랜딩 푸터에 `v{버전}` 상시 표시.
- **OG 소셜 프리뷰**(`opengraph-image.tsx`·`twitter-image.tsx`): 링크를 펼칠 때 보이는
  1200×630 미리보기를 `next/og` ImageResponse로 코드 생성(에셋 0개 §11) — 아이콘과 같은
  마스코트 도트(`mascot-cells.ts`, pwa-icon과 공유)를 크게 얹는다. **기본은 영어**(크롤러가
  읽는 정본). `layout.tsx`에 `metadataBase`(env `NEXT_PUBLIC_SITE_URL`/`VERCEL_URL`)와
  `openGraph`·`twitter` 추가. 언어별 동적 OG는 `/og?lang=..`(`og/route.tsx`)가 같은
  제너레이터로 만든다(§2). 픽셀 폰트는 싣지 않고(빌드 안정성 §12) 마스코트 도트가 정체성 담당.
- **공유하기**(`share-button.tsx`): 랜딩의 버튼 — `navigator.share`가 있으면 모바일 OS 공유
  시트, 없으면 링크 클립보드 복사 폴백. 공유 문구는 현재 언어(§2). 취소·미지원은 조용히(§12).
- 접근성: 장식 요소에 `aria-hidden`, 포커스 링(focus-visible) 유지, HTML 텍스트로 정보 전달.
- 이 저장소는 **프론트엔드 초급자 학습용**이다: 모든 코드에 "왜"를 설명하는 꼼꼼한 한글 주석.
  코드를 바꾸면 관련 주석·이 문서도 함께 갱신. 문법 해설은 `docs/` 학습 문서
  시리즈가 담당한다 (주석은 "왜", docs는 "어떻게 읽나").

## 15. 튜닝 가이드

게임 밸런스를 바꿀 때는 코드 로직이 아니라 **상수부터** 만진다:
- 손맛·판정·성장·조이스틱·연료: `src/app/play/joops-game.tsx`의 `TUNE`
- 난이도 곡선: `joops-game.tsx`의 `spawn()`(확률·속도)과 `update()`(스폰 간격) 안의 수식
- 분위기(색·대사·종류): `src/lib/constants.ts`

수치를 바꾸면 이 문서의 해당 표/수식도 갱신할 것.
