# 02. 타입스크립트 문법 — 꺾쇠와 물음표의 정체

타입스크립트(TS)는 **자바스크립트에 "타입 설명서"를 붙인 언어**예요.
실행 전에 "여긴 숫자만 와야 하는데 문자열이 왔어요!" 하고 잡아 줍니다.
이 챕터에서는 우리 코드에 실제로 나오는 TS 문법을 전부 훑어요.

## 1. 기본: 타입 주석 (`: 타입`)

```ts
// src/lib/debris.ts
export function stepJunk(j: Junk, dt: number): void {
```

- `j: Junk` — "j는 Junk 모양의 객체여야 해요"
- `dt: number` — "dt는 숫자여야 해요"
- `): void` — "이 함수는 돌려주는 값이 없어요" (void = 없음)

## 2. type — 객체의 설계도

```ts
// src/lib/debris.ts
export type Junk = {
  kind: JunkKind;
  x: number;
  y: number;
  eatT: number;   // -1이면 평소, 0 이상이면 "꿀꺽" 중
  // ...
};
```

`type 이름 = { ... }`으로 "이 모양의 객체"라는 설계도를 만듭니다.
이후 `const j: Junk = ...`라고 쓰면 TS가 모양이 맞는지 검사해 줘요.
오타(`j.knid` 같은)를 치는 순간 에디터가 빨간 줄을 그어 줍니다.

## 3. 유니온 타입 (`|`) — "이것들 중 하나"

```ts
// src/app/play/joops-game.tsx
type Phase = "title" | "playing" | "over";
```

`|`는 "또는"입니다. Phase는 딱 저 세 문자열 중 하나만 될 수 있어요.
`phase = "gameover"`라고 쓰면 (오타!) 즉시 에러 — 이게 유니온의 힘입니다.

낙하물 종류도 같은 방식이에요:

```ts
// src/lib/constants.ts
export type JunkKind =
  | "satellite" | "bolt" | "can" | "spring"
  | "glove" | "toolbag" | "fairing" | "cubesat"
  | "hazard" | "fuel" | "star"
  | "magnet" | "slowmo" | "shield";
```

첫 줄부터 `|`로 시작하는 건 그냥 줄맞춤용 스타일이에요. 의미는 같습니다.

## 4. `as const` — "이 값은 앞으로도 정확히 이 값"

```ts
// src/app/play/joops-game.tsx
const TUNE = {
  startR: 24,
  maxR: 38,
  // ...
} as const;
```

`as const`를 붙이면 TS가 `startR`을 그냥 `number`가 아니라 **정확히 24**라는
타입으로 기억하고, 전체를 읽기 전용으로 만듭니다. "튜닝 상수는 아무도 실수로
못 바꾼다"는 보증이죠.

부작용도 하나 있어요 — 우리 코드에 이런 주석이 있는 이유입니다:

```ts
// number 명시: as const 리터럴 타입(3000)이 그대로 옮으면 재대입이 막힌다
let fuel: number = TUNE.maxFuel;
```

`TUNE.maxFuel`의 타입은 "정확히 3000"이라서, 그냥 `let fuel = TUNE.maxFuel`이라
쓰면 fuel엔 3000 말고 다른 수를 넣을 수 없게 돼요. 그래서 `: number`라고
"넓은 타입"을 명시해서 풀어 줍니다.

## 5. `Record`와 `Partial` — 객체를 만드는 도우미 타입

```ts
// src/lib/constants.ts — "모든 JunkKind 각각에 string 값이 있다"
export const JUNK_COLORS: Record<JunkKind, string> = {
  satellite: "#8ecbff",
  // ...14종 전부 안 쓰면 에러! 새 종류를 추가하면 여기도 채우라고 TS가 알려줘요
};

// src/lib/storage.ts — "일부 JunkKind에만 number 값이 있어도 된다"
export type Inventory = Partial<Record<JunkKind, number>>;
```

- `Record<K, V>`: K의 **모든** 항목을 키로 갖는 객체. 빠뜨리면 에러.
- `Partial<...>`: 모든 키를 "있어도 되고 없어도 되는" 것으로 바꿈.
  인벤토리는 아직 못 먹은 종류가 있을 수 있으니 Partial이 딱 맞죠.

`Record<K, V>`처럼 꺾쇠 안에 타입을 넣는 것을 **제네릭**이라고 불러요 —
"타입을 인자로 받는 타입"입니다.

## 6. `?.`(옵셔널 체이닝)과 `??`(널 병합)

```ts
// src/app/play/joops-game.tsx
const ctx = canvas?.getContext("2d");     // canvas가 null이면 그냥 undefined
onTogglePause={() => pauseRef.current?.()} // 함수가 있으면 호출, 없으면 무시

// src/app/bag/bag-grid.tsx
const count = inv[kind] ?? 0;   // 값이 null/undefined면 0을 대신 사용
```

- `a?.b` — a가 null/undefined면 터지지 않고 조용히 undefined.
- `a ?? b` — a가 null/undefined일 때만 b. (`||`와 달리 0이나 ""는 그대로 살려요!)

## 7. `null` / `undefined`를 상태 표현에 활용하기

우리 코드가 즐겨 쓰는 3단계 패턴입니다:

```ts
// src/app/play/leaderboard.tsx
// undefined = 불러오는 중, null = 실패(목록 숨김), 배열 = 표시
const [runs, setRuns] = useState<RunRow[] | null | undefined>(undefined);
```

"없음"에도 종류가 있어요 — **아직 없음(undefined)**과 **실패해서 없음(null)**.
UI가 LOADING을 보여줄지, 조용히 숨길지를 이 차이로 결정합니다.

## 8. 배열 타입과 `readonly`

```ts
// src/lib/constants.ts
export const JUNK_FOOD_KINDS = [
  "satellite", "bolt", /* ... */
] as const;
```

배열에 `as const`를 붙이면 **readonly 튜플**(내용·순서까지 고정)이 됩니다.
그런데 readonly 배열의 `.includes()`는 타입이 까다로워서, 게임 코드에서는
이렇게 잠깐 "넓은 타입으로 봐 달라"고 캐스팅해요:

```ts
// src/app/play/joops-game.tsx
(JUNK_FOOD_KINDS as readonly string[]).includes(j.kind)
```

`as`는 "내가 책임질게, 이 타입으로 취급해 줘"라는 뜻입니다. 남용은 금물!

## 9. 화살표 함수와 콜백

```ts
const pushUi = () => setUi({ ... });          // 인자 없는 한 줄 함수
junks.filter(isFood)                          // 함수를 "값처럼" 넘기기
rows.map((r, i) => ({ name: r.name, ... }))   // 객체를 바로 돌려줄 땐 ( ) 필수!
```

마지막 줄이 중요해요 — `=> { ... }`라고 쓰면 중괄호가 "함수 본문"으로 해석되니,
객체를 돌려주려면 `=> ({ ... })`처럼 괄호로 감쌉니다.

## 10. 구조 분해와 스프레드

```ts
let { w, h } = fitCanvas(canvas);       // 객체에서 w, h를 꺼내 변수로
({ w, h } = fitCanvas(canvas));         // 이미 선언된 변수에 다시 채울 땐 괄호 필요

sparks.push(...makeSparks(x, y, c, 7)); // 배열을 "낱개로 펼쳐서" 넣기
setUi({ ...ui, score: 10 });            // 객체 복사 + 일부만 바꾸기
<GameUi {...ui} pet={pet} />            // JSX에서: ui의 모든 필드를 props로 전달
```

## 11. 템플릿 리터럴 (백틱 문자열)

```ts
makePopup(`COMBO x${mult}!`, ...)       // 문자열 안에 ${변수} 삽입
`${URL}/rest/v1/pets?select=...`        // 주소 조립에도 씁니다
```

## 12. async / await — "기다렸다가 다음 줄"

```ts
// src/lib/leaderboard.ts
export async function fetchTopTotals(...): Promise<TotalRow[] | null> {
  const res = await fetch(...);   // 응답이 올 때까지 기다림
  if (!res.ok) return null;
  const rows: unknown = await res.json();
  return Array.isArray(rows) ? (rows as TotalRow[]) : null;
}
```

- `async` 함수는 항상 **Promise**(미래의 값 약속)를 돌려줍니다.
- `await`는 그 약속이 지켜질 때까지 기다렸다가 값을 꺼내요.
- `Promise.all([a, b])`는 여러 개를 **동시에** 기다립니다 — 랭킹 페이지가
  단판·누적을 한꺼번에 가져올 때 쓰죠.
- `unknown`은 "정체를 모르니 검사 후에 쓰겠다"는 가장 안전한 타입이에요.
  바로 위 코드처럼 `Array.isArray`로 확인한 뒤에야 `as`로 좁힙니다.

## 13. try / catch — 실패해도 게임은 계속

```ts
// src/lib/storage.ts
export function loadBest(): number {
  try {
    const raw = localStorage.getItem(BEST_KEY);
    // ...
  } catch {
    return 0;      // 시크릿 모드 등에서 localStorage가 터져도 게임은 산다
  }
}
```

`try` 블록에서 에러가 나면 프로그램이 죽는 대신 `catch`로 점프합니다.
이 프로젝트의 대원칙 "부가 기능이 게임을 죽이면 안 된다"를 실현하는 문법이에요.
