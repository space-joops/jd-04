# 08. 스타일링 — className에 붙은 암호 해독

## 1. Tailwind CSS — 유틸리티 클래스라는 발상

보통 CSS는 클래스 이름을 짓고(`.title-box`) 따로 스타일을 씁니다.
**Tailwind**는 반대로, 미리 만들어진 "한 가지 일만 하는" 클래스를 조합해요:

```tsx
<div className="flex flex-col items-center gap-4 px-6 text-center">
```

| 클래스 | CSS로 치면 | 뜻 |
|---|---|---|
| `flex` | `display: flex` | 유연한 배치 상자 |
| `flex-col` | `flex-direction: column` | 세로로 쌓기 |
| `items-center` | `align-items: center` | 교차축(가로) 가운데 |
| `justify-between` | `justify-content: space-between` | 양 끝으로 밀기 |
| `gap-4` | `gap: 1rem` | 자식들 사이 간격 (숫자 × 0.25rem) |
| `px-6` / `pt-3` | padding 좌우 / 위 | 안쪽 여백 |
| `mt-auto` | `margin-top: auto` | 남은 공간을 위 여백으로 → 바닥에 붙음 |
| `text-3xl` / `text-xs` | font-size | 글자 크기 |
| `text-white/60` | 흰색, 투명도 60% | `/숫자`는 투명도 |
| `border-2 border-white/30` | 테두리 두께·색 | |
| `hidden` · `md:text-6xl` | 반응형 | `md:`는 "화면이 768px 이상일 때만" |

HUD 한 줄을 해독해 볼까요:

```tsx
<div className="flex items-start justify-between px-5 pt-3 text-3xl tracking-widest">
```

"가로 flex, 위 정렬, 양 끝 배치(점수 왼쪽·하트 오른쪽), 좌우 여백 5,
위 여백 3, 큰 글자, 자간 넓게." — 읽는 법만 알면 CSS 파일을 오갈 필요가 없어요.

## 2. 위치 잡기 — absolute / fixed / inset-0

```tsx
<div className="fixed inset-0">                {/* 화면 전체에 고정 */}
<canvas className="absolute inset-0 h-full w-full touch-none" />
<div className="pointer-events-none absolute inset-0">  {/* 캔버스 위 HUD */}
```

- `fixed`: 뷰포트(화면) 기준 고정 — 스크롤해도 그 자리.
- `absolute`: 가장 가까운 `relative` 조상 기준.
- `inset-0`: `top/right/bottom/left: 0` — 꽉 채우기.
- 게임 화면 = "fixed로 꽉 채운 캔버스 + 그 위에 absolute HUD" 샌드위치 구조입니다.

`z-10`, `z-50` 같은 `z-index`는 겹침 순서 — 숫자가 클수록 위에 옵니다
(랜딩 콘텐츠 z-10, 인트로 오버레이·갱신 토스트 z-50).

## 3. 픽셀 폰트 두 벌 — 그리고 한글 문제

```css
/* src/app/globals.css — Tailwind 4의 @theme 확장 */
@theme {
  --font-pixel: "Press Start 2P", monospace;
  --font-pixel-ko: "Galmuri11", "Press Start 2P", monospace;
}
```

이렇게 등록하면 `font-pixel`, `font-pixel-ko` 클래스가 생깁니다.
**Press Start 2P에는 한글 글리프가 없어서** 캔버스와 영문 UI는 `font-pixel`,
한글 문구(스토리·도감 등)는 `font-pixel-ko`(Galmuri)를 씁니다.
폰트는 CSS `@import`로 불러오는데, 오프라인이면 조용히 폴백 폰트로 넘어가요 —
next/font를 안 쓰는 이유입니다 (네트워크 없으면 빌드가 통째로 실패하거든요).

레트로 간판 느낌의 그림자는 CSS로:

```tsx
style={{ textShadow: "4px 4px 0 #000" }}   // 번짐 0의 "하드 섀도" = 픽셀 감성
```

## 4. 애니메이션 — Tailwind 내장 + 직접 만든 keyframes

내장 유틸리티:

```tsx
className="animate-pulse"    // 깜빡깜빡 (TAP TO START)
className="animate-bounce"   // 통통 (NEW BEST!)
```

직접 만든 것 — 스타워즈 오프닝 크롤:

```css
/* src/app/globals.css */
@keyframes sw-crawl {
  from { transform: rotateX(24deg) translateY(100%); }   /* 화면 아래에서 */
  to   { transform: rotateX(24deg) translateY(-170%); }  /* 우주 저편으로 */
}
.starwars-stage {
  perspective: 340px;      /* 원근감 — 이게 있어야 rotateX가 "눕는" 효과가 됨 */
  overflow: hidden;
  mask-image: linear-gradient(to bottom,
    transparent 0%, black 28%, black 96%, transparent 100%);
    /* 위쪽은 서서히 투명 — 글이 멀어지며 사라지는 연출 */
}
.starwars-crawl {
  transform-origin: 50% 100%;               /* 회전축을 바닥에 */
  animation: sw-crawl 18s linear forwards;  /* 18초, 등속, 1회 재생 후 유지 */
}
```

- `@keyframes 이름 { from {...} to {...} }` — 시작과 끝 상태를 정의.
- `animation: 이름 시간 속도곡선 반복` — 적용.
- `forwards` — 끝나면 끝 상태 유지 (그리고 JS의 `onAnimationEnd`로 인트로가 스스로 닫혀요).

접근성 배려도 CSS로 합니다:

```css
@media (prefers-reduced-motion: reduce) {
  .starwars-crawl { animation: none; transform: none; }
}
```

OS에서 "동작 줄이기"를 켠 사용자에게는 크롤 대신 정지 화면을 보여줍니다.

## 5. 모바일 디테일

```css
/* globals.css의 body */
user-select: none;                        /* 드래그가 텍스트 선택으로 변하는 것 방지 */
-webkit-tap-highlight-color: transparent; /* 탭할 때 회색 번쩍임 제거 */
overscroll-behavior: none;                /* 당겨서 새로고침 방지 */
```

```tsx
{/* 노치(카메라 구멍) 밑으로 HUD가 숨지 않게 */}
style={{ paddingTop: "env(safe-area-inset-top)" }}
```

`env(safe-area-inset-*)`는 브라우저가 알려주는 "노치·홈바 안전 여백"이에요.
`max(4rem, env(...))`처럼 조합하면 "최소 4rem, 노치가 크면 그만큼 더"가 됩니다.

## 6. 인라인 style을 쓰는 순간들

Tailwind로 다 하지 않고 `style={{ ... }}`을 쓰는 곳이 있어요:

```tsx
style={{ color: COLORS.accent, borderColor: COLORS.accent }}
```

색이 **자바스크립트 상수**(`constants.ts` 팔레트)에서 오기 때문입니다.
"색은 전부 constants.ts에서만"이라는 프로젝트 규칙을 지키려면, 클래스에
색을 하드코딩하는 대신 상수를 인라인으로 주입하는 게 맞아요 — 규칙이
문법 선택을 결정하는 좋은 예입니다.
