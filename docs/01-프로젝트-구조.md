# 01. 프로젝트 구조 — 파일이 왜 거기에 있을까

## Next.js가 뭔가요?

우리가 쓰는 **Next.js**는 리액트(React) 위에 "웹사이트를 만들 때 늘 필요한 것들"
(주소별 페이지, 빌드, 최적화)을 얹은 프레임워크예요. 핵심 규칙 하나만 기억하세요:

> **`src/app` 폴더의 구조가 곧 웹사이트의 주소(URL) 구조다.**

| 파일 | 주소 | 역할 |
|---|---|---|
| `src/app/page.tsx` | `/` | 랜딩(첫) 페이지 |
| `src/app/play/page.tsx` | `/play` | 게임 본편 |
| `src/app/rank/page.tsx` | `/rank` | 랭킹 페이지 |
| `src/app/bag/page.tsx` | `/bag` | 인벤토리(도감) |
| `src/app/layout.tsx` | (모든 주소) | 모든 페이지를 감싸는 공통 틀 |

폴더를 만들고 그 안에 `page.tsx`를 두면 새 페이지가 생깁니다. 끝!
이것이 Next.js의 **App Router** 방식이에요.

## 서버 컴포넌트 vs 클라이언트 컴포넌트

Next.js의 컴포넌트는 두 종류입니다.

- **서버 컴포넌트** (기본값): 서버에서 HTML을 만들어 보냅니다.
  버튼 클릭 반응 같은 "동작"은 못 넣지만, 빠르고 가벼워요.
- **클라이언트 컴포넌트**: 파일 맨 위에 `"use client"`라고 쓰면
  브라우저에서 실행됩니다. `useState` 같은 훅과 이벤트 처리는 여기서만 가능해요.

실제 예를 볼까요? 랜딩 페이지(`src/app/page.tsx`)는 글과 링크뿐이라 서버 컴포넌트,
그 배경에서 마스코트가 노는 캔버스(`src/app/attract-sky.tsx`)는 애니메이션이
필요하니 클라이언트 컴포넌트입니다:

```tsx
"use client";           // ← 이 한 줄이 "브라우저에서 실행해 주세요"라는 선언

import { useEffect, useRef } from "react";
```

**구분하는 요령**: `useState`, `useEffect`, `onClick` 이 셋 중 하나라도 쓰면
`"use client"`가 필요합니다. 서버 컴포넌트가 클라이언트 컴포넌트를 자식으로
품는 건 자유예요 — 랜딩(서버)이 `<AttractSky />`(클라이언트)를 품듯이.

## 우리 프로젝트의 큰 그림

```
src/
├── app/        페이지들 (주소 = 폴더 구조)
│   └── play/   게임 본편 — joops-game.tsx가 심장
└── lib/        페이지가 아닌 "도구" 모음
    ├── constants.ts   색·이름·종류 (분위기의 원본)
    ├── debris.ts      낙하물의 데이터·물리·그림
    ├── mascot.ts      주인공 그리기
    ├── sound.ts       소리 합성
    └── storage.ts     localStorage 저장
```

왜 `lib`을 따로 뒀을까요? **"그리는 코드"와 "게임 규칙 코드"를 분리**하기
위해서예요. 예를 들어 마스코트의 생김새를 바꾸고 싶으면 `mascot.ts` 하나만
건드리면 됩니다. 게임 본체(`joops-game.tsx`)는 "마스코트를 여기에 그려 줘"라고
부탁만 하고, 어떻게 생겼는지는 몰라요. 이런 분리를 **관심사의 분리**라고 부릅니다.

## import — 파일끼리 도구를 빌려 쓰는 법

```tsx
// src/app/play/joops-game.tsx 맨 위
import { fitCanvas } from "@/lib/canvas";
import { drawMascot } from "@/lib/mascot";
```

- `import { 이름 } from "경로"` — 다른 파일이 `export`한 것을 가져옵니다.
- `@/`는 "src 폴더에서부터"라는 별명이에요 (`tsconfig.json`에 설정돼 있음).
  `../../lib/canvas`처럼 상대 경로를 세는 것보다 훨씬 읽기 쉽죠.

`export`가 붙은 것만 밖에서 쓸 수 있습니다:

```ts
// src/lib/canvas.ts
export function fitCanvas(...) { ... }   // ← 밖에서 쓸 수 있음
const 내부헬퍼 = ...;                     // ← 이 파일 전용
```

## metadata — 페이지의 명함

서버 컴포넌트 파일에서 `metadata`를 export하면 Next.js가 브라우저 탭 제목,
검색엔진 설명 등을 채워 줍니다:

```tsx
// src/app/rank/page.tsx
export const metadata: Metadata = {
  title: "SPACE JOOPS · RANKING",
  description: "우주 청소부 순위표 — 단판 최고 기록과 통산 수거량",
};
```

## 특별한 파일들

- `src/app/layout.tsx` — 모든 페이지의 공통 껍데기. `<html>`, `<body>`가 여기 있어요.
- `src/app/manifest.ts` — PWA(홈 화면 설치) 정보를 만드는 특수 파일. (→ 06챕터)
- `src/app/apple-icon.tsx` — iOS 아이콘을 **코드로 그려서** 제공. (→ 06챕터)
- `public/` — 주소로 바로 접근되는 정적 파일. `public/sw.js`는 `/sw.js`가 됩니다.
