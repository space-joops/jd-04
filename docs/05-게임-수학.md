# 05. 게임 수학 — Math가 만드는 손맛

수학이라고 겁먹지 마세요. 이 게임의 수학은 딱 네 가지입니다:
**거리, 감쇠, 파도(sin), 확률.** 하나씩 게임 속 실제 쓰임새로 봅니다.

## 1. 거리 — 충돌 판정의 전부

두 점 사이 거리는 피타고라스 정리인데, JS가 함수로 줍니다:

```ts
const dist = Math.hypot(mascot.x - j.x, mascot.y - j.y);
// = √((x차이)² + (y차이)²)
```

마스코트도 낙하물도 "원"으로 취급해요. 두 원이 닿았다 = 중심 거리가
반지름 합보다 작다:

```ts
// 먹이 획득 (§7 황금률: 획득은 후하게)
if (dist < mascot.r + j.size * TUNE.eatJudge + bonus) eat(j);
// 가시 피격 (피격은 박하게 — 그림보다 작은 판정)
if (dist < mascot.r + j.size * TUNE.hitJudge) ...
```

`eatJudge = 0.65`, `hitJudge = 0.75` 같은 배율이 "아슬아슬하게 먹었다!"와
"억울하게 죽지 않았다"를 만드는 손맛 조절 나사입니다.

## 2. 지수 감쇠 — 부드러운 따라가기의 만능 공식

이 게임에서 가장 자주 나오는 한 줄:

```ts
pos += (target - pos) * Math.min(1, dt * k);
```

"목표까지 남은 거리의 일부만큼 다가간다"예요. 매 프레임 남은 거리가
일정 비율로 줄어드니, 처음엔 빠르고 가까워질수록 살살 — 자연스러운
미끄러짐이 공짜로 나옵니다. k가 클수록 즉각 반응해요.

쓰이는 곳: 시선(gaze), 입벌림, 타이틀 마스코트 복귀, "꿀꺽" 빨려들기…

**속도 감쇠**도 같은 원리입니다 (우주 관성):

```ts
vx -= vx * TUNE.friction * dt;   // 속도가 매 초 일정 비율씩 준다
```

추진을 멈춰도 바로 서지 않고 스르륵 미끄러지는 이유예요.

## 3. sin — 둥둥, 살랑살랑, 반짝반짝

`Math.sin(각도)`은 -1과 1 사이를 부드럽게 오가는 파도입니다.
시간을 넣으면 "왕복 운동"이 공짜:

```ts
// 타이틀 화면에서 둥둥 (1.6 = 빠르기, 10 = 진폭 px)
mascot.y = h * 0.72 + Math.sin(elapsed * 1.6) * 10;

// 낙하물의 좌우 살랑거림 — x는 항상 "중심축 x0 + 파도"로 재계산
j.x = j.x0 + Math.sin(j.swayT) * j.sway;

// 별의 맥박 (크기가 ±12% 숨쉬기)
const pulse = 1 + 0.12 * Math.sin(j.swayT * 5);

// 배경 별 반짝임 — 별마다 속도·위상이 달라야 같이 안 깜빡인다
0.55 + 0.45 * Math.sin(t * 속도 + 위상);
```

낙하물 코드에 "x를 직접 옮기지 말고 x0를 옮겨라"라는 경고 주석이 있는
이유도 여기 있어요 — x는 매 프레임 파도 공식으로 **다시 계산되는 파생값**이라,
x를 옮겨 봐야 다음 프레임에 도로 튕겨 돌아옵니다. 자석 효과가 `j.x0 +=`인 이유!

## 4. 확률 — Math.random으로 만드는 운명

`Math.random()`은 0 이상 1 미만의 무작위 수. 확률 추첨은 이렇게 합니다:

```ts
// src/lib/debris.ts — pickKind: 순서대로 주사위를 굴린다
if (Math.random() < 0.07) return "star";        // 7%
if (Math.random() < 0.04) return 파워업;         // 4%
if (Math.random() < 0.06) return "fuel";        // 6%
if (Math.random() < hazardChance) return "hazard";
return 쓰레기8종[Math.floor(Math.random() * 8)]; // 나머지 균등
```

- `Math.random() < 0.07` — 7% 확률로 참.
- `Math.floor(Math.random() * n)` — 0 ~ n-1의 정수 (배열에서 무작위 뽑기).

범위 랜덤도 자주 나와요:

```ts
const size = 13 + Math.random() * 6;          // 13 ~ 19
spawnTimer = base * (0.7 + Math.random() * 0.6);  // ±30% 지터
```

스폰 간격의 ±30% 지터는 명세서가 "불규칙성도 설계다"라고 못박은 부분 —
정확히 일정한 리듬은 외워져서 지루해지거든요.

### 결정론적 유사난수 — 랜덤인데 매번 같아야 할 때

배경 별은 무작위처럼 보이지만 **매 프레임 같은 자리**에 있어야 해요.
Math.random을 매 프레임 부르면 별이 지지직거리며 온 화면을 뜁니다. 그래서:

```ts
// src/lib/backdrop.ts — 같은 i를 넣으면 언제나 같은 값이 나오는 "해시"
function hash(i: number): number {
  const x = Math.sin(i * 127.1 + 311.7) * 43758.5453;
  return x - Math.floor(x);   // 소수부만 취하면 0~1의 "랜덤스러운" 수
}
```

sin을 크게 튀긴 뒤 소수부만 남기는 고전 트릭이에요. "씨앗이 같으면 결과가
같은" 가짜 난수 — 별 46개의 위치·크기·반짝임 속도가 전부 이걸로 고정됩니다.

## 5. 난이도 곡선 — 점수에 비례해서

```ts
const difficulty = Math.min(1, score / 400);           // 0 → 1 (400점에 최고)
const speedScale = 1 + difficulty * 1.1;               // 낙하 최대 2.1배
const hazardChance = 0.1 + 0.13 * difficulty;          // 가시 10% → 23%
const base = Math.max(0.42, 1.05 - score / 900);       // 스폰 간격, 바닥 0.42초
```

패턴이 보이나요? `Math.min`/`Math.max`로 **상한·하한**을 만들고, 그 사이를
점수에 비례해 움직입니다. "잘할수록 어려워지되, 무한히 어려워지진 않게".

## 6. 콤보 배율 — 나눗셈 내림의 활용

```ts
// 5개마다 배율 +1, 최대 ×5
const comboMult = () =>
  Math.min(TUNE.comboMaxMult, 1 + Math.floor(combo / TUNE.comboStep));
```

`Math.floor(combo / 5)`는 0~4개→0, 5~9개→1, 10~14개→2… 계단 함수예요.
거기에 1을 더하면 ×1, ×2, ×3…이 되고, `Math.min`이 ×5 천장을 만듭니다.

## 7. 벡터 정규화 — 방향만 남기기

조이스틱 입력(dx, dy)에서 "방향"만 뽑아 힘을 곱하는 부분:

```ts
const dist = Math.hypot(dx, dy);
vx += (dx / dist) * speed * dt;   // dx/dist = 길이 1로 만든 방향 성분
vy += (dy / dist) * speed * dt;
```

벡터를 자기 길이로 나누면 길이가 1인 "순수한 방향"이 됩니다(정규화).
거기에 원하는 속도를 곱하면 어느 방향이든 같은 힘으로 밀 수 있어요.
시선(gaze)도, 자석 끌림도 전부 같은 패턴입니다.

## 8. 각도 — atan2와 라디안

```ts
ctx.rotate(Math.atan2(dy, dx));   // 추진 방향을 바라보게 회전
```

`Math.atan2(y, x)`는 벡터의 각도(라디안)를 줍니다. 캔버스 회전도, 방패의
궤도 도트(`Math.cos(a)`, `Math.sin(a)`로 원 위의 점 찍기)도 라디안 세상이에요.
한 바퀴 = 2π(약 6.28)만 기억하면 됩니다.
