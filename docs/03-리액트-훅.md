# 03. 리액트 훅 — use로 시작하는 함수들의 정체

리액트 컴포넌트는 "상태가 바뀌면 화면을 다시 그리는 함수"예요.
**훅(hook)**은 그 함수 안에서 상태·수명 같은 능력을 빌려 쓰는 도구이고,
전부 `use`로 시작합니다. 우리 게임에 나오는 훅을 하나씩 봅시다.

## 1. useState — 기억하고, 바뀌면 다시 그린다

```tsx
// src/app/play/leaderboard.tsx
const [tab, setTab] = useState<Tab>("run");
```

- `tab` — 현재 값 (처음엔 `"run"`)
- `setTab` — 값을 바꾸는 유일한 방법. **호출하면 컴포넌트가 다시 렌더링**됩니다.
- `<Tab>` — 꺾쇠로 "이 상태의 타입"을 알려줄 수 있어요.

버튼과 연결하면:

```tsx
<button onClick={() => setTab("total")}>누적 TOP 5</button>
```

### 함수로 초기값 만들기 (lazy 초기화)

```tsx
// src/app/play/joops-game.tsx
const [soundOn, setSoundOn] = useState(() => {
  const m = loadMuted();   // localStorage 읽기 — 첫 렌더 때 딱 한 번만 실행
  setMuted(m);
  return !m;
});
```

`useState(값)` 대신 `useState(() => 값)`을 쓰면 함수가 **첫 렌더 때 한 번만**
실행돼요. localStorage처럼 조금 비싼 일을 매 렌더마다 반복하지 않게 됩니다.

### 이전 값 기반으로 바꾸기

```tsx
setSoundOn((prev) => !prev);   // "지금 값의 반대로" — 안전한 토글 방법
```

## 2. useEffect — 렌더 후에 할 일 + 뒷정리

**"컴포넌트가 화면에 나타난 뒤에 이걸 실행해 줘"**가 useEffect입니다.
게임 전체가 사실 하나의 거대한 useEffect예요:

```tsx
// src/app/play/joops-game.tsx (뼈대만)
useEffect(() => {
  // ① 마운트 직후 한 번 실행: 캔버스 준비, 게임 상태 만들기, 루프 시작
  const canvas = canvasRef.current;
  let raf = requestAnimationFrame(frame);
  canvas.addEventListener("pointerdown", onPointerDown);

  // ② 돌려주는 함수 = "정리 함수": 컴포넌트가 사라질 때 실행
  return () => {
    cancelAnimationFrame(raf);
    canvas.removeEventListener("pointerdown", onPointerDown);
    disposeAudio();
  };
}, []);   // ③ 의존성 배열이 [] = "딱 한 번만"
```

세 가지 포인트:

1. **본문**: 마운트(화면에 등장) 후 실행됩니다.
2. **정리 함수**: `return () => {...}` — 언마운트 때 실행. 안 하면 페이지를
   떠나도 게임 루프가 유령처럼 계속 돕니다. (진짜로 그런 버그가 흔해요!)
3. **의존성 배열**: `[]`이면 한 번만. `[pet, score]`처럼 값을 넣으면
   "그 값이 바뀔 때마다" 정리 후 재실행됩니다.

의존성이 있는 실제 예 — 게임오버 리더보드:

```tsx
// src/app/play/leaderboard.tsx
useEffect(() => {
  let alive = true;   // ← 언마운트 뒤 도착한 응답을 무시하는 안전핀
  (async () => {
    const status = await submitResult(pet, score, eaten);
    if (alive && status !== "ok") setSendError(status);
    // ...
  })();
  return () => { alive = false; };
}, [pet, score, eaten]);
```

`alive` 패턴을 눈여겨보세요. 네트워크 응답은 언제 올지 모르는데, 그 사이
사용자가 재시작해서 컴포넌트가 사라졌다면? 정리 함수가 `alive = false`로
바꿔 둔 덕분에 "죽은 컴포넌트의 setState" 문제를 피합니다.

또, `useEffect(async () => ...)`는 금지라서(정리 함수 자리가 꼬여요)
`(async () => { ... })();`라는 **즉시 실행 함수**로 감싸는 관용구를 씁니다.

## 3. useRef — 다시 그리지 않고 기억만

```tsx
const canvasRef = useRef<HTMLCanvasElement>(null);
// ...
<canvas ref={canvasRef} ... />
```

`useRef`는 `.current`라는 주머니 하나짜리 상자예요. 두 가지 용도로 씁니다:

1. **DOM 잡기**: `ref={canvasRef}`라고 걸어두면 `canvasRef.current`가
   실제 `<canvas>` 요소가 됩니다.
2. **렌더와 무관한 값 보관**: 바뀌어도 다시 그릴 필요 없는 값.

두 번째 용도의 재미있는 예 — React 버튼이 게임 클로저 내부 함수를 부르는 다리:

```tsx
// src/app/play/joops-game.tsx
const pauseRef = useRef<(() => void) | null>(null);
// useEffect 안 (게임 클로저): 함수를 상자에 넣어 둔다
pauseRef.current = () => setPaused(!paused);
// 렌더 쪽: 버튼이 상자 속 함수를 꺼내 부른다
<GameUi onTogglePause={() => pauseRef.current?.()} ... />
```

일시정지 여부는 게임 루프의 지역 변수인데, HUD 버튼은 React 세계에 있죠.
ref가 두 세계를 잇는 우편함 역할을 합니다.

## 4. 대원칙 — "초당 60번 변하는 것은 캔버스에"

게임 상태(주인공 위치, 낙하물 배열…)를 전부 useState로 만들면 어떻게 될까요?
초당 60번 setState → 초당 60번 리렌더 → 게임이 아니라 벤치마크가 됩니다.

그래서 우리 게임은:

```tsx
useEffect(() => {
  // 게임 상태는 전부 "클로저 지역 변수" — React는 모른다
  let score = 0;
  let junks: Junk[] = [];
  const mascot = { x, y, r };

  // React에는 "보여줄 값이 실제로 바뀔 때만" 알린다
  const pushUi = () => setUi({ phase, score, hearts, ... });
```

- 매 프레임 변하는 것 → 클로저 변수 + 캔버스가 직접 그림
- 가끔 변하는 것(점수·하트·phase) → `pushUi()`로 React에 통보 → HUD 갱신

**클로저**란 "함수가 자기가 태어난 곳의 변수를 계속 기억하는 것"이에요.
`update`, `draw`, `eat` 같은 함수들이 전부 같은 `score`, `junks`를 공유하는
이유가 바로 이 useEffect 안에서 함께 태어났기 때문입니다.

## 5. useSyncExternalStore — React 바깥 세상 구독하기

```tsx
// src/app/best-score.tsx
const best = useSyncExternalStore(subscribe, loadBest, () => 0);
```

localStorage는 React가 모르는 "외부 저장소"죠. 이 훅은 세 가지를 받습니다:

1. `subscribe` — "바뀌면 알려줘" 등록 (storage 이벤트 리스너)
2. `loadBest` — 현재 값을 읽는 법
3. `() => 0` — 서버에서 렌더링할 때의 값 (서버엔 localStorage가 없으니까)

덕분에 다른 탭에서 게임을 해 최고 기록이 갱신되면, 랜딩의 숫자도 따라 바뀝니다.

## 6. 조건부 렌더링 — 화면을 상황에 따라

```tsx
{phase === "title" && ( <div>...타이틀...</div> )}     // 조건이 참일 때만
{newBest ? <p>NEW BEST!</p> : <p>BEST: {best}</p>}     // 둘 중 하나
{rows.map((r, i) => <li key={i}>...</li>)}             // 목록은 map + key
```

`key`는 리액트가 "이 줄이 아까 그 줄인지" 알아보는 이름표예요. 목록을
렌더링할 때 반드시 붙입니다.
